- 패키지
  - 패키지는 관련 클래스를 분류하는 방법이다. 프로그램의 복잡성을 제거한다.
- import
  - 동일한 패키지에 있는 클래스를 사용할 때는 경로를 입력하지 않아도 된다. 하지만 패키지가 다를때는 경로를 입력해야 한다.
  - 클래스의 모든 경로를 기억하기 어렵고 불편하다. Java는 `import`문을 사용하여 다른 패키지에 클래스를 불러올 수 있다.
  - 특정 패키지 클래스를 여러개 사용할 때 `*` 와일드 카드 문자를 사용하여 `import`문을 사용할 수 있다.
- 규칙
  - 패키지 이름은 클래스와 인터페이스 이름과 충돌을 방지하기 위해 소문자를 사용한다.
  - `io.github.devmeeple` 같은 도메인 역순으로 패키지를 만든다.
  - 예약어로 등록된 키워드는 패키지 이름으로 사용할 수 없다. `_`을 사용하면 문제를 해결할 수 있다.
