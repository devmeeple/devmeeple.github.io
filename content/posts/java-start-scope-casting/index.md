- 표현범위가 큰 타입에서 작은 타입으로 변환할때 문제가 발생한다. Java의 기본 규칙에 어긋나기 때문에 책임지지 않는다. 개발자의 명시적 형변환이 필요하다.
  - 명시적형변환은 잔돈을 안받는것과 비슷하다.
- 형변환은 변수의 값을 직접적으로 변경하지 않는다. 읽은 값을 변경한다.
- 같은 타입의 결과는 같은 타입이다. 반면 다른 타입의 결과는 표현범위가 큰 타입으로 자동형변환이 발생한다.
